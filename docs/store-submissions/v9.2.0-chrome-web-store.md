## 4. Permission Justifications

**No changes from v9.0.0.** The manifest permission set is identical. Reproduced verbatim below for reviewer convenience.

### Required Permissions

#### `history`

Core functionality. SmrutiCortex indexes browser history to provide instant search. Uses `chrome.history.search()` to query history entries and `chrome.history.onVisited` to index new visits in real time as the user browses.

#### `bookmarks`

Enables unified search across history and bookmarks. Uses `chrome.bookmarks.getTree()` to index bookmarks alongside history so users can find any saved page from a single search box.

#### `storage`

Persists user preferences and extension state locally. Uses `chrome.storage.local` to store settings (search options, AI configuration, display preferences), cached favicon data, recent search queries, and performance metrics. All data stays on device.

#### `tabs`

Opens search results in browser tabs and communicates with the quick-search overlay. Uses `chrome.tabs.query()` to find the active tab, `chrome.tabs.create()` to open results, `chrome.tabs.sendMessage()` to communicate with the content script overlay, and `chrome.tabs.update()` to switch to existing tabs matching search results.

#### `alarms`

Keeps the MV3 service worker responsive. Uses `chrome.alarms.create()` for periodic keepalive pings that prevent cold-start delays when the user presses the keyboard shortcut. Without this, the first search after idle would be slow because Chrome terminates inactive service workers.

#### `scripting`

Re-injects the quick-search content script after Chrome auto-updates the extension. When Chrome updates an extension, all previously injected content scripts are invalidated — the keyboard shortcut (Ctrl+Shift+S) stops working on already-open tabs. Uses `chrome.scripting.executeScript()` to re-inject ONLY our own content script (`quick-search.js`) into the active tab so the shortcut resumes without requiring a page reload. Never executes arbitrary or remote code. Only triggered when the user presses the keyboard shortcut.

#### `activeTab`

Grants temporary host permission for ONLY the current active tab when the user presses the keyboard shortcut (Ctrl+Shift+S). This temporary permission is required by `chrome.scripting.executeScript()` to re-inject our content script after an extension update. No data is read from the page. Permission expires immediately when the user navigates away or switches tabs. Strictly user-initiated — never granted automatically.

#### `sessions`

Displays recently closed tabs in the tab-switcher mode (`@` prefix) so users can find and restore pages they accidentally closed. Uses `chrome.sessions.getRecentlyClosed()` to list up to 10 recently closed tabs and `chrome.sessions.restore()` to reopen a selected tab. This directly extends the extension's core purpose of helping users find pages in their browsing history — recently closed tabs are the most common "lost page" scenario.

#### `windows`

Focuses the correct browser window when the user switches to a tab from search results. When a search result belongs to a tab in another window, `chrome.windows.update(windowId, { focused: true })` brings that window to the foreground. Also powers "Move Tab to New Window" and "Merge Windows" command palette actions. All uses are strictly user-initiated.

### Optional Permissions

These three permissions are declared as `optional_permissions` in the manifest. They are never granted at install time. They are only requested at runtime via `chrome.permissions.request()` when the user explicitly enables the "Advanced Browser Commands" setting. If the user denies the permission prompt, the feature stays off and these APIs are never called.

#### `tabGroups`

Powers command palette actions to create, collapse, expand, name, and color tab groups. Every call to `chrome.tabGroups.query()` and `chrome.tabGroups.update()` is gated behind a `chrome.permissions.contains()` check that verifies the permission was granted. If the user never enables Advanced Browser Commands, this API is never invoked.

#### `browsingData`

Powers command palette actions to clear browser cache, cookies, local storage, form data, downloads history, and saved passwords. Every call to `chrome.browsingData.removeCache()`, `chrome.browsingData.removeCookies()`, and related methods is gated behind a `chrome.permissions.contains()` check. If the user never enables Advanced Browser Commands, this API is never invoked.

#### `topSites`

Shows the browser's most-visited sites in the command palette. Uses `chrome.topSites.get()` to retrieve the list, gated behind a `chrome.permissions.contains()` check. If the user never enables Advanced Browser Commands, this API is never invoked.

### Optional Host Permissions

#### `<all_urls>`

Declared as `optional_host_permissions`. Used in two ways: (1) `content_scripts` match pattern to inject the quick-search overlay on any webpage when the user presses the keyboard shortcut (Ctrl+Shift+S), and (2) optional favicon fetching from Google's favicon API for display purposes only. No page content is collected, transmitted, or stored. All data remains on the user's device.

## 7. Changes from Previous Submission

### New in v9.1.0 (vs v9.0.0) — Reliability + Observability

This is a maintenance release focused on runtime reliability, observability, and release-infrastructure hardening. **No new permissions, no new user-facing features that require reviewer attention.**

**Runtime reliability:**
- Service-worker cold start — register `onMessage` listener synchronously so the very first message after SW hibernation is received reliably
- Quick-search overlay — fix blank overlay regression caused by stale port / port rate-limiting edge cases
- Popup hardening — eliminate `Unchecked runtime.lastError: message port closed` warnings via proper `lastError` callbacks on fire-and-forget messages
- Favicon hardening — fallback SVG for empty favicon URLs; graceful handling of context invalidation during favicon fetch
- Embedding pipeline — correct 0-dimension embedding semantics (no longer treated as valid); skip redundant embedding-processor restart when already completed; remove session-cap busy-wait
- Log-noise reduction in AI / embedding paths

**Observability (developer-facing, no runtime impact for users):**
- AOP-inspired tracing framework with `@Traced` decorator and `traced()` wrapper, applied to embedding, search, cache, and database modules
- Unified Troubleshooter for self-healing diagnostics

**Release infrastructure (no runtime impact for users):**
- `verify.mjs` — run-all-and-report script combining lint, tests, coverage, build, and E2E in a single invocation
- Coverage threshold gated in pre-commit hook
- Bundle-size benchmark gate in release preflight
- `lint:strict` script for zero-warning CI runs
- E2E slow-mo npm aliases (`test:e2e:slow`, etc.) for reliable cross-platform debugging
- `.gitattributes` enforcing LF line endings — eliminates phantom diffs on Windows/macOS

**Test suite:** 1,233 unit tests across 46 files + 45 E2E Playwright tests across 7 specs (unchanged from v9.0.0; existing tests updated to cover the reliability fixes above).

**What reviewers do NOT need to re-check:**
- Permission set — byte-identical to v9.0.0 manifest (`history`, `bookmarks`, `storage`, `tabs`, `alarms`, `scripting`, `activeTab`, `sessions`, `windows` required; `tabGroups`, `browsingData`, `topSites` optional; `<all_urls>` optional host)
