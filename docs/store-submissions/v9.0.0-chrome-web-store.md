## Permission Justifications

### `history`

Core functionality. SmrutiCortex indexes browser history to provide instant search. Uses `chrome.history.search()` and `chrome.history.onVisited` to access and monitor history entries.

### `bookmarks`

Core functionality. Enables unified search across history AND bookmarks. Uses `chrome.bookmarks.getTree()` to index bookmarks for search.

### `storage`

Required for settings persistence. Uses `chrome.storage.local` to store user preferences (log level, AI settings, display options) and cached search data.

### `tabs`

Required to open search results in tabs and communicate with content scripts. Uses `chrome.tabs.query()` to find active tab, `chrome.tabs.sendMessage()` for overlay communication.

### `alarms`

Required to keep service worker responsive. Uses `chrome.alarms.create()` for periodic keepalive pings to prevent cold start delays on keyboard shortcuts.

### `scripting`

Required for zero-downtime extension updates. After the extension is updated by Chrome, the keyboard shortcut (Ctrl+Shift+S) stops working on already-open tabs because the old content script is invalidated. Uses `chrome.scripting.executeScript()` to re-inject ONLY our own content script (`quick-search.js`) into the active tab so the shortcut resumes without requiring a page reload. Never executes arbitrary or remote code. Only triggered when the user presses the keyboard shortcut — never runs automatically in the background.

**Code reference:** `src/background/service-worker.ts` → `reinjectContentScript()` function.

### `activeTab`

Required in combination with the `scripting` permission. Grants temporary host permission for ONLY the current active tab when the user presses the keyboard shortcut (Ctrl+Shift+S). This temporary permission is needed by `chrome.scripting.executeScript()` to re-inject our content script after an extension update. No data is read from the page. Permission expires immediately when the user navigates away or switches tabs. Strictly user-initiated — never granted automatically.

**Code reference:** `src/background/service-worker.ts` → `reinjectContentScript()` function, called from `chrome.commands.onCommand` listener.

### Host permission (`<all_urls>`)

The `<all_urls>` host permission is used in two ways: (1) `content_scripts` match pattern to inject the quick-search overlay on any page when the user presses the keyboard shortcut, and (2) optional favicon fetching from Google's favicon API for display purposes only. No page content is collected, transmitted, or stored beyond locally-cached favicons. All data remains on the user's device.

### Optional API permissions (`tabGroups`, `browsingData`, `topSites`)

Declared as **`optional_permissions`** in the manifest; **not** granted at install.

**Advanced Browser Commands** (Settings → General) is **off by default**. Enabling it triggers **`chrome.permissions.request()`** for `tabGroups`, `browsingData`, and `topSites` only. If the user **denies** the prompt, the feature **stays off** and those APIs are unused. This is **separate** from optional host access (`<all_urls>`), which applies to other optional features (e.g. favicons) and is not requested by this toggle.

**Use:** Tab group management (`tabGroups`), browsing data cleanup (`browsingData`), Top Sites–related commands (`topSites`). All are user-initiated behind explicit opt-in.
