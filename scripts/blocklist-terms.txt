# ──────────────────────────────────────────────────────────────────────────────
# SmrutiCortex Repository Blocklist
# ──────────────────────────────────────────────────────────────────────────────
# Purpose: Prevent accidental re-introduction of company- / product-specific
# identifiers into this open-source repository. All examples, tests, docs,
# and source doc-comments must use synthetic / RFC-2606 placeholders
# (example.com, example.org, etc.) and neutral fictional product names.
#
# Enforced by:
#   - scripts/check-blocklist.mjs         (pre-commit hook + CI entry point)
#   - .github/workflows/blocklist.yml     (blocks PRs that reintroduce terms)
#
# Line format
# ──────────────────────────────────────────────────────────────────────────────
#   hash:<length>:<sha256-hex-64>   HARD-blocked term. The literal is NEVER
#                                   stored in the repo — only a salted SHA-256
#                                   hash. The scanner hashes candidate
#                                   substrings from scanned files and probes
#                                   this set. This means anyone reading this
#                                   file cannot see which strings are blocked.
#
#   soft:<literal>                  SOFT-blocked term. Allowed inside product
#                                   source under src/**/*.{ts,tsx,html,css}
#                                   (and generated coverage reports), blocked
#                                   in docs / tests / user-facing surfaces.
#                                   These literals are NOT sensitive — they
#                                   are generic integration names that must
#                                   exist in the code to work at all.
#
# Empty lines and lines starting with '#' are ignored.
#
# Adding a new hard term
# ──────────────────────────────────────────────────────────────────────────────
# Do NOT type the literal into this file. Instead run:
#
#     node scripts/hash-blocklist-term.mjs "<term>"          # argv form
#     node scripts/hash-blocklist-term.mjs --stdin           # safer: no shell-history trace
#
# then paste the printed `hash:...` line into the HARD section below. The
# literal only exists locally in your shell / stdin; git only ever sees
# the hash. Bumping the salt in `check-blocklist.mjs` invalidates every
# entry below and requires re-hashing.
#
# Self-exempt files (never scanned — would trivially trip themselves):
#   - scripts/blocklist-terms.txt           (this file)
#   - scripts/check-blocklist.mjs           (the scanner)
#   - scripts/hash-blocklist-term.mjs       (the helper)
#   - scripts/__tests__/check-blocklist.test.mjs
#
# Governance
# ──────────────────────────────────────────────────────────────────────────────
# Any PR that modifies this list must be reviewed by a CODEOWNER.
# Removing a hard hash requires written justification in the PR description.
# ──────────────────────────────────────────────────────────────────────────────
